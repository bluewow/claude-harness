[01]

워크플로우 소개

이 문서는 Claude Code의 슬래시 커맨드, 백그라운드 훅 및 동적 대시보드를 연계하여 비즈니스 기획을 즉시 분석 가능한 구조의 프로덕션 코드로 전환해내는 자동화 파이프라인 지침서입니다. 복잡한 마케팅 요소를 배제하고 오직 규격, 파일 구조, 인터페이스 표준에만 집중합니다.

💡

모든 킷은 .claude/ 내부 파일들의 약속된 전제와 정적 해석 룰을 따라 작동합니다. 최종 산출물은 프로젝트 루트의 output/ 폴더로 격리되어 정렬됩니다.

[03]

기획 및 개발 명령어 레퍼런스

Command 실행 방식 주요 입력값 핵심 산출 파일 사용 용도
/init Foreground — CLAUDE.md 프로젝트 파일 스캔 및 Claude Code 전용 표준 컨텍스트 구축
/pm [요청] Foreground (대화형) 자연어 기획안 brief.md, *.html 타겟 세그먼트, 화면 설계 질의응답을 거쳐 완결된 기획서 도출
/dev [ID] Background 태스크 ID complete.md 작성 완료된 brief.md를 로드하여 소스 코드 자동 연산 구현 및 병합
/do [작업] Background 버그 패치 등 자연어 지시 complete.md 기획 및 HTML 시안 전개 과정 없이 코드 수정 및 에러 검증을 다이렉트 수행
[04]

Next.js 전용 기동 명령어

새 프로젝트 초기 가동 및 구조적 룰셋 검증을 담당하는 전용 슬래시 커맨드 사양입니다.

실행 패턴: /create-nextjs [프로젝트명] 프로덕션 전용 사양

빌드 보일러플레이트를 엔터프라이즈 사양(Tailwind 4, Zustand 상태 통합, React Query, Pretendard 폰트 페어링, TS Strict 모드 정렬)에 맞추어 14단계 연속 공정으로 즉시 배포합니다.

- Next.js 기반의 표준 View-Action-Container (VAC) 레이어 자동 수립
- 404, error, loading 화면 구성 포함
- ESLint / Prettier 통합 빌드 최종 완결성 점검
실행 패턴: /check-nextjs [--report | --fix] 검사 및 치유

현재 활성화된 프레임워크가 아키텍처 및 테마 룰셋(Global CSS 변수 무결성, 공급자 인젝션 등) 10대 검사 항목을 정상 통과하는지 자가 채점합니다.

--report : 자동 조치 없이 위반 구조 리포트만 기록
--fix : 누락된 레이아웃 복원 및 빌드 차단 에러 최대 3회 자가 해결 시도

commands/pm.md

기획 워크플로우 명세

사용자 질의를 설계 템플릿과 정적으로 결합하여 brief.md 결과물을 구성합니다. 이 파일은 요구사항을 명세 형태로 정형화하는 데 사용됩니다.

* 왼쪽 트리의 임의 파일을 선택하면 실시간 메타 데이터와 역할 서술이 연계 동기화됩니다.
