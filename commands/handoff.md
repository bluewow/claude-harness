## 경로 규약

`{PROJECT_ROOT}` = `${CLAUDE_PROJECT_DIR}` → 없으면 cwd 에서 위로 올라가며 `.claude/` 가 있는 디렉토리.
핸드오프 파일 = `{PROJECT_ROOT}/.output/handoff.md` (**1 파일**). 파일 안에 **세션별 블록**(`## 🧵 <slug>`)을 여러 개 보관하고, 상단에 **세션 인덱스 표**를 둔다.

> 멀티탭 안전: 각 탭(세션)은 **자기 세션 블록만** 롤링 갱신하고 다른 세션 블록은 건드리지 않는다. 단, 두 탭이 **동시에** 저장하면 파일 단위 Read→Write 가 겹쳐 한쪽이 유실될 수 있으니, 가급적 저장은 탭마다 시점을 살짝 어긋나게 한다.

## 원칙: handoff ≠ memory

- **handoff** = 지금 작업의 **현재 위치**(이번 세션 한 일·진행 중·다음 스텝). 롤링, 다음 세션에 갱신/폐기.
- **memory**(`memory/*.md` + `MEMORY.md`, 매 세션 자동 로드) = 오래 가는 **사실**(컨벤션·결정·취향). 무기한.
- 판별: "내일·모레도 사실인가?" → memory / "지금 어디까지 했나?" → handoff.
- 둘이 겹치면 **memory 우선**, handoff 엔 링크만. 저장 중 오래 갈 사실이 보이면 memory 승격을 **제안**(자동 저장 금지).

## 세션 키(slug)

- 각 세션 블록은 영문 kebab-case 짧은 슬러그로 식별(예: `ia-renewal`, `staff-sync`, `annual-leave`).
- `--name "..."` 가 있으면 그 값을 슬러그화해 키로 사용.
- 없으면 **현재 작업 주제**에서 짧은 슬러그를 추론.

## 모드 분기

`$ARGUMENTS` 파싱:

| 인자 | 모드 |
|:----|:----|
| `resume` 포함 | **이어가기** → Step R |
| `list` 포함 | **목록만 보기** → Step L |
| `delete` / `rm` / `clean` 포함 | **세션 삭제** → Step D |
| 그 외(없음 / `--name` / `--next` 만) | **저장**(기본) → Step S |

---

## Step S: 저장 (기본)

1. **세션 키 결정** — `--name` 있으면 슬러그화, 없으면 현재 작업 주제로 추론.
2. **이번 세션 정리** — 대화 맥락에서 추출: 한 일(변경·결정), 현재 상태(되는 것/진행 중/막힌 것), 검증 결과(typecheck·lint·build), 다음 할 일, 열린 결정/블로커.
   - 보강(선택): git 하위 모듈이 있으면 `git -C web status -s` / `git -C web diff --stat` 로 변경 파일 확인(이 레포 루트는 git 아님).
3. `{PROJECT_ROOT}/.output/handoff.md` 가 있으면 `Read`:
   - **자기 세션 블록(`## 🧵 <slug>`) 찾기.**
     - `--name` 없이 추론한 경우, 슬러그가 정확히 같지 않아도 **주제가 같은 기존 세션 블록**이 있으면 그걸로 본다(드리프트로 인한 중복 생성 방지). 애매하면 *"기존 `<slug>` 세션 갱신 / 새 세션 생성?"* 1회 확인.
   - **매칭 블록이 있으면 그 블록만 롤링 갱신**:
     - "이번 세션 한 일"은 **이번 세션 내용으로 교체**. 이전 내용 중 아직 안 끝난 것은 "다음 할 일"로 내리고, 나머지는 버림. 헤더의 업데이트 날짜·검증 상태 갱신.
     - **다른 세션 블록은 절대 건드리지 않는다.**
   - **매칭 블록이 없으면** 새 `## 🧵 <slug>` 블록을 골격대로 추가.
   - 상단 **세션 인덱스 표**에서 해당 행(업데이트 날짜·다음 할 일 요약) 갱신/추가.
4. 없으면 `{PROJECT_ROOT}/.output/` 생성 후 골격(인덱스 표 + 이 세션 블록 1개)으로 새로 작성.
5. `--next "..."` 가 있으면 그 항목을 **해당 세션 "다음 할 일" 최상단**에 추가.
6. 오래 갈 사실(컨벤션·결정)이 보이면 "memory 로 승격할까요?" 한 줄 제안.
7. 채팅에 짧게 보고: 저장 위치 + **세션 키** + 다음 할 일 Top 3.

## Step R: 이어가기 (resume)

1. `{PROJECT_ROOT}/.output/handoff.md` `Read`. **없으면** "핸드오프 없음 — `/handoff` 로 먼저 저장하세요" 안내 후 종료.
2. **세션 인덱스 표**를 목록으로 출력(번호·slug·제목·업데이트·다음 할 일 1줄).
   ```
   🗂️ 핸드오프 세션
   1) ia-renewal   IA 현행화            (2026-06-30)  다음: …
   2) staff-sync   직원 데이터 실시간 동기화  (2026-06-29)  다음: …
   ```
3. 이어갈 세션 선택:
   - 세션이 **4개 이하면** `AskUserQuestion` 으로 선택지 제시.
   - 더 많으면 "이어갈 세션의 번호 또는 slug 를 입력하세요" 로 받기.
4. 선택한 세션 블록 + 자동 로드된 `MEMORY.md` + 관련 `memory/*.md` 를 결합해 컨텍스트 정리:
   ```
   📋 이어가기 — <slug>
   - 현재 상태: …
   - 다음 할 일: 1) …  2) …  3) …
   - 열린 결정/블로커: …
   ```
5. `AskUserQuestion` 1회 — `1순위로 시작 / 다른 항목 선택 / 지금은 대기`.
6. 선택대로 진행. 구현 작업이면 `/task` 로 위임 가능. 진행 시작 시 해당 세션 블록 상태를 "진행 중"으로 본다(완료 후 `/handoff` 로 갱신 권장).

## Step L: 목록 보기 (list)

1. `Read`. 없으면 안내 후 종료.
2. **세션 인덱스 표**를 그대로 출력(번호·slug·제목·업데이트·다음 할 일 1줄). 더 진행 안 함.

## Step D: 세션 삭제 (delete / rm / clean)

1. `Read`. 없으면 안내 후 종료.
2. 세션 목록 출력(번호·slug·제목·업데이트).
3. 삭제 대상 선택(**다중 가능**):
   - 세션이 **4개 이하면** `AskUserQuestion` `multiSelect: true`.
   - 더 많으면 "삭제할 세션의 번호/slug 를 쉼표로 입력" 으로 받기.
4. 삭제 직전 **확인 1회**(어떤 세션을 지우는지 명시).
5. 확인되면 해당 `## 🧵 <slug>` 블록 + 상단 인덱스 표의 해당 행을 제거. **나머지 세션은 그대로 보존.**
6. 남은 세션 수 보고. 0개가 되면 골격(빈 인덱스 표)만 남긴다.

## handoff 문서 골격 (새로 생성할 때)

```markdown
# Handoff — {프로젝트명}

> 멀티 세션 핸드오프 (1 파일 · 세션별 블록). 업데이트는 각 세션 블록에 표기.
> handoff 는 '현재 작업 위치'만 — 오래 갈 사실은 memory(MEMORY.md).

## 🗂️ 세션 인덱스
| # | 세션(slug) | 제목 | 업데이트 | 다음 할 일(요약) |
|:--|:--|:--|:--|:--|
| 1 | {slug} | {제목} | {YYYY-MM-DD} | … |

---

## 🧵 {slug} — {제목}
> 업데이트 {YYYY-MM-DD} · 검증 typecheck {✓/✗/–} lint {✓/✗/–} build {✓/✗/–}

### 🎯 현재 상태
- 진행 중: …
- 막힌 것 / 주의: …

### ✅ 이번 세션 한 일
- … (파일은 `경로:라인` 또는 링크)

### ⏭️ 다음 할 일 (우선순위)
1. … (바로 착수 가능한 단위로, 필요한 맥락 한 줄 포함)
2. …

### ❓ 열린 결정 / 블로커
- … (유저 입력이 필요한 것)

### 📌 이어서 하려면
- 핵심 파일/위치: …
- 실행: …
- 관련 문서·메모리: docs/… , memory/…
```

> 세션 블록이 2개 이상이면 블록 사이를 `---` 로 구분한다.
