## 경로 규약

`{PROJECT_ROOT}` = `${CLAUDE_PROJECT_DIR}` → 없으면 cwd 에서 위로 올라가며 `.claude/` 가 있는 디렉토리.
핸드오프 파일 = `{PROJECT_ROOT}/.output/handoff.md` (롤링 1장 — 항상 최신 상태만 유지).

## 원칙: handoff ≠ memory

- **handoff** = 지금 작업의 **현재 위치**(이번 세션 한 일·진행 중·다음 스텝). 롤링, 다음 세션에 갱신/폐기.
- **memory**(`memory/*.md` + `MEMORY.md`, 매 세션 자동 로드) = 오래 가는 **사실**(컨벤션·결정·취향). 무기한.
- 판별: "내일·모레도 사실인가?" → memory / "지금 어디까지 했나?" → handoff.
- 둘이 겹치면 **memory 우선**, handoff 엔 링크만. 저장 중 오래 갈 사실이 보이면 memory 승격을 **제안**(자동 저장 금지).

## 모드 분기

`$ARGUMENTS` 파싱:

| 인자 | 모드 |
|:----|:----|
| `resume` 포함 | **이어가기** → Step R |
| 그 외(없음 / `--next "..."` 만) | **저장**(기본) → Step S |

---

## Step S: 저장 (기본)

1. **이번 세션 정리** — 대화 맥락에서 추출: 한 일(변경·결정), 현재 상태(되는 것/진행 중/막힌 것), 검증 결과(typecheck·lint·build), 다음 할 일, 열린 결정/블로커.
   - 보강(선택): git 하위 모듈이 있으면 `git -C web status -s` / `git -C web diff --stat` 로 변경 파일 확인(이 레포 루트는 git 아님).
2. `--next "..."` 가 있으면 그 항목을 **"다음 할 일" 최상단**에 추가.
3. `{PROJECT_ROOT}/.output/handoff.md` 가 있으면 `Read`:
   - 아래 골격에 맞춰 **부분 갱신**(전체 재작성 금지). 변경된 섹션만.
   - "이번 세션 한 일"은 **이번 세션 내용으로 교체**(롤링). 이전 내용 중 아직 안 끝난 것은 "다음 할 일"로 내리고, 나머지는 버림.
   - 헤더의 업데이트 날짜·검증 상태 갱신.
4. 없으면 `{PROJECT_ROOT}/.output/` 생성 후 골격으로 새로 작성.
5. 오래 갈 사실(컨벤션·결정)이 보이면 "memory 로 승격할까요?" 한 줄 제안.
6. 채팅에 짧게 보고: 저장 위치 + 다음 할 일 Top 3.

## Step R: 이어가기 (resume)

1. `{PROJECT_ROOT}/.output/handoff.md` `Read`. **없으면** "핸드오프 없음 — `/handoff` 로 먼저 저장하세요" 안내 후 종료.
2. 자동 로드된 `MEMORY.md` + 관련 `memory/*.md` 와 함께 읽어 컨텍스트 결합.
3. 채팅에 짧게 정리:
   ```
   📋 이어가기
   - 현재 상태: …
   - 다음 할 일: 1) …  2) …  3) …
   - 열린 결정/블로커: …
   ```
4. `AskUserQuestion` 1회 — `1순위로 시작 / 다른 항목 선택 / 지금은 대기`.
5. 선택대로 진행. 구현 작업이면 `/task` 로 위임 가능. 진행 시작 시 handoff 의 해당 항목 상태를 "진행 중"으로 본다(완료 후 `/handoff` 로 갱신 권장).

## handoff 문서 골격 (새로 생성할 때)

```markdown
# Handoff — {프로젝트명}

> 업데이트 {YYYY-MM-DD} · 검증 typecheck {✓/✗/–} lint {✓/✗/–} build {✓/✗/–}
> handoff 는 '현재 작업 위치'만 — 오래 갈 사실은 memory(MEMORY.md).

## 🎯 현재 상태
- 진행 중: …
- 막힌 것 / 주의: …

## ✅ 이번 세션 한 일
- … (파일은 `경로:라인` 또는 링크)

## ⏭️ 다음 할 일 (우선순위)
1. … (바로 착수 가능한 단위로, 필요한 맥락 한 줄 포함)
2. …

## ❓ 열린 결정 / 블로커
- … (유저 입력이 필요한 것)

## 📌 이어서 하려면
- 핵심 파일/위치: …
- 실행: …
- 관련 문서·메모리: docs/… , memory/…
```
