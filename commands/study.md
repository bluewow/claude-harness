바로 직전에 네가 한 작업(또는 대화에서 다룬 결과)을 다시 설명해 줘. $ARGUMENTS 가 있으면 그 부분만 집중해서 설명해.

## 설명 규칙

- **12살에게 설명하듯이**: 쉬운 단어를 쓰고, 전문용어가 꼭 필요하면 바로 옆에 한 줄로 풀어 써 줘. (예: "캐시(한 번 계산한 걸 적어두는 메모장)")
- **결과만 말하지 말 것**: 어떻게 그 결과에 도달했는지 **중간 과정을 순서대로** 보여 줘. 각 단계마다 ① 무엇을 했는지 ② 왜 그렇게 했는지 ③ 그래서 무엇이 나왔는지를 말해 줘.
- **직접적이고 구체적으로**: 두루뭉술한 비유만 늘어놓지 말고, 실제 파일 이름·실제 값·실제 코드 조각을 예로 들어 설명해. 비유는 개념을 처음 소개할 때 한 번만.
- **건너뛴 판단도 설명**: "왜 다른 방법이 아니라 이 방법이었는지" 갈림길이 있었다면 그 선택 이유도 말해 줘.

## 출력 형식

1. **한 줄 요약** — 결국 무슨 일이 일어난 건지 한 문장.
2. **차근차근 과정** — 단계별로 번호를 붙여서. 각 단계는 "무엇을 → 왜 → 결과" 흐름으로.
3. **새로 나온 말 풀이** — 설명 중 등장한 용어를 2~5개만 골라 한 줄씩.
4. **스스로 확인해 보기** — 이해했는지 확인할 수 있는 간단한 질문 1개 (답도 같이, 접어서 아래에).

새로운 작업을 하거나 코드를 고치지 말고, 설명만 해.
