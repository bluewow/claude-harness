# 앱 룩 목업 (mockup)

**목적: UI·IA 결정은 텍스트로 고르지 않는다.** 실제 앱과 같은 룩의 폰 프레임 목업을 만들어 시각 비교로 결정하고, 확정 내역을 같은 문서에 누적한다(결정 이력이 곧 시안 문서). 사용자 선호: **가장 단순한 안을 기본 추천으로**.

## 산출물 · 위치 (3곳 동기화)

| 사본 | 위치 | 역할 |
|---|---|---|
| 작업본 | 스크래치패드 `{주제}-mockup.html` | Edit 대상 — 전체 재생성 금지, 부분 수정 |
| 웹 게시본 | Artifact | 사용자가 보는 화면 — **같은 file_path 재게시 = 같은 URL 유지**, `label` 은 `vN-요약`, favicon 은 주제당 1개 고정 |
| 보존본 | `docs/시안-{주제}-목업.html` | 세션이 끝나도 남는 정본 — 게시할 때마다 `cp` 로 동기화 |

새 세션에서 기존 목업을 이어 갱신할 때: 보존본을 스크래치패드로 복사해 작업하고, Artifact 는 사용자에게 기존 URL 을 받아 `url` 파라미터로 재게시(안 그러면 새 URL 이 생긴다).

## 프로세스

1. **토큰 대조** — 프레임 안 색·radius 는 `web/src/app/globals.css` 가 SoT, 컴포넌트 합성 규칙은 `docs/design.md`. 템플릿의 oklch 값은 스냅샷이므로 **다르면 코드가 승리**(템플릿도 같은 작업에서 갱신).
2. **화면 스펙은 PRD §A 기반** — 요구·정책(특히 법적 제약)을 프레임에 반영하고 주석 점으로 표시. 정책 문서가 없으면 먼저 product-docs intake.
3. **프레임 작성** — `templates/frame-library.html` 의 클래스만 조합(새 클래스가 필요하면 라이브러리에 추가). 내용은 실제 같은 예시 데이터(가상의 매장·인명), 실존 브랜드 금지. 도메인 문구 규율(예: 채용의 "추천·매칭" 금지)은 해당 PRD §A.4 를 따른다.
4. **결정이 필요한 것만 A/B** — 갈림길이 진짜일 때만 `.pair` 섹션으로 2안 나란히(3안 이상 금지 — 단순 선호). A안 = 가장 단순한 안 + `추천` pill, 각 안에 `.why` 근거 3줄 이내.
5. **게시 + 결정 요청** — Artifact 게시 후 채팅에서 결정 번호로 질문. AskUserQuestion 이 아니라 **목업을 보고 고르게** 한다.
6. **확정 반영** — 채택 pill `추천 → 채택 ✓`, footer 진행 현황 갱신, **PRD 에 결정 기록**(product-docs — §A.4 정책 또는 수용 기준), 보존본 재복사. 구현이 끝나면 footer 를 "구현 완료"로.

## 문서 구성 규격

- **헤더**: eyebrow(`매장비서 — 화면 시안`) · h1(`{주제} 목업 vN`) · lede(근거 PRD·대기 중 결정 요약) · meta 칩(도메인·모드·토큰 준수) · legend(주석 점 범례)
- **FLOW 섹션**: `sec-eyebrow`(FLOW N — 대상) + h2 + sec-note(플로우 한 줄) + `.grid` 에 `.shot` 나열
- **`.shot`** = `.phone` 프레임 + `h3`(화면 ID · 이름) + `.sub`(한 줄 설명) + `.notes`(주석 리스트)
- **화면 ID**: 목업 로컬 ID — 역할별 접두(예: 구직자 J*, 사장 O*, 알림톡 K*). IA-현행화의 S번호와 별개이며, 구현 시 IA 에서 정식 S번호를 받는다.
- **결정 섹션**: `sec-eyebrow`(결정 N — 주제) + `.pair` 에 `.option`(B안) / `.option.reco`(A안+pill)
- **footer**: 진행 현황 한 문단 — 채택된 결정·구현 상태·남은 결정. 갱신 시점마다 여기만 고쳐도 이력이 읽힌다.

## 주석 점 규칙 (프레임 아래 `.notes`)

- **앰버 점**(`.dot.a`) — 법·정책 제약이 만든 요소 (예: "매장명 표시 필수(법정)", "연락처 비노출 — 공리 20"). 근거(법·공리·PRD 조항)를 괄호로.
- **그린 점**(`.dot.g`) — 제품 설계 포인트 (패턴 재사용, 전환 장치, 버전 표기 `(vN)`)

## 프레임 재현 규칙 (design.md ↔ 라이브러리 클래스)

| 앱 규칙 | 라이브러리 클래스 |
|---|---|
| 리스트 행(카드+아바타+상태) | `.pcard.rowline` + `.avatar` + `.rmain` + `.badge` |
| StatusBadge 3종 | `.badge.ok / .warn / .mut` — warning 은 "행동 필요"에만 |
| 주의 배너 | `.wbanner` + `.wicon` + `.wtext` |
| 하단 고정 CTA(1개) | `.cta-bar` + `.btn`(보조는 `.btn.ghost`) |
| 바텀시트(lg+ 다이얼로그) | `.dim` + `.sheet` + `.grab` |
| 세그먼트 컨트롤 | `.seg` + `span.on` |
| 필터/조건 칩 | `.chip`(기본) `.chip.on`(액션) `.chip.set`(활성 조건) |
| 폼 필드·검증 에러 | `.field` + `.input`(`.err` + `.hint.dest`) |
| 토글 | `.tgl` |
| 프로모션 히어로 | `.promo` + `.fp` |
| 데스크톱(lg+) 홈 | `.phone.pc` + `.pcnav` + `.pcmain` + `.statrow/.stat` |
| 카카오 요소 | `.btn.kakao` · 알림톡 채팅 프레임은 `docs/시안-채용-목업.html` 의 K1 프레임(인라인 스타일) 참조 |

아이콘: 목업 HTML 에서는 lucide 룩의 미니 인라인 SVG 허용(앱 코드의 "인라인 SVG 금지"는 목업에 미적용). 프레임은 **항상 앱 라이트 룩**(라이트 우선 프로젝트), 페이지 크롬만 라이트/다크 대응.

## 금지

- ❌ 문서 전체 재생성 — 항상 부분 Edit 후 재게시
- ❌ 결정 없는 A/B 남발 — 갈림길이 없으면 단일안
- ❌ 프레임 안 임의 hex — 라이브러리 변수(globals.css 스냅샷) 경유
- ❌ 확정된 결정을 다음 버전에서 지우기 — pill·footer 로 이력 유지
