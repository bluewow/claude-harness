# 노무히어로 docs 라우팅

**목적: 늘어나는 `docs/*.md` 를 매번 통째로 읽지 않고도, 사용자 질문을 적절한 문서로 라우팅해서 인용 기반으로 정확하게 답한다.**

핵심 원칙:
- 문서 본문을 통째로 컨텍스트에 넣지 않는다. 라우팅 → grep → 매치 ±20줄 read.
- 답변에는 **반드시 `파일명:줄번호` 인용**을 포함한다. 인용 못 붙이면 추측이므로 "문서에서 확인 못 함" 으로 명시.
- 라우팅 맵은 `web/CLAUDE.md` §"문서 갱신" 의 매핑과 1:1 동기. 충돌 시 CLAUDE.md 가 SoT.

## 답변 절차

1. **라우팅** — 위 표에서 우선/보조 파일 1~2개 선택.
2. **Grep** — 사용자 질문에서 명사·전문용어 추출 → `Grep pattern="..." path="docs/<선택파일>" output_mode=content -n=true` 로 위치 찾기.
3. **부분 Read** — 매치된 줄 ±20줄만 `Read offset=... limit=...` 로 읽는다. **전체 파일 읽기 금지**(예외: 파일이 200줄 미만일 때만).
4. **합성** — 인용한 줄과 그 의미를 사용자 언어로 1~3문단으로 정리. 표/리스트가 답에 맞으면 그렇게.
5. **인용** — `파일명:줄번호` 형태로 출처 표시. 예: `docs/PRD-급여계산.md:142`.
6. **갭 표시** — 문서에 해당 정책이 없으면 추측하지 말고 "문서에서 확인되지 않음. 코드(`web/src/...`) 기준 또는 사용자 결정 필요" 로 솔직히 안내.

- 새 PRD 추가/분할·문서 파일명 변경 시: 이 SKILL.md 의 라우팅 표를 같은 PR 에서 갱신.
