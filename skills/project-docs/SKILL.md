# 노무히어로 docs 라우팅

**목적: `docs/*.md` 를 통째로 읽지 않고, 질문을 문서로 라우팅해 인용 기반으로 정확하게 답한다.**

문서 목록·매핑의 단일 SoT 는 **`web/CLAUDE.md` §"문서 갱신"** (문서 체계 표 + 갱신 매핑 표). 이 스킬은 자체 라우팅 표를 갖지 않는다 — 매 세션 로드되는 그 표를 그대로 쓴다.

## 답변 절차

1. **라우팅** — CLAUDE.md 문서 체계·갱신 매핑 표에서 질문 주제에 맞는 파일 1~2개 선택. 화면·라우트 질문은 `IA-현행화.md`, 플로우 도식이 필요하면 `docs/flows/*.html` 보조.
2. **Grep** — 질문의 명사·전문용어로 `Grep pattern="..." path="docs/<파일>" output_mode=content -n=true`.
3. **부분 Read** — 매치 ±20줄만 Read. **전체 파일 읽기 금지**(200줄 미만 파일만 예외).
4. **답변** — `파일명:줄번호` 인용 필수. 문서에 없으면 추측하지 말고 "문서에서 확인되지 않음. 코드(`web/src/...`) 기준 또는 사용자 결정 필요"로 명시.

## 활성화 금지

- 코드 디버깅("버튼 동작 안 해") — Grep/Read 로 직접 처리
- 디자인·구현 작업 — `design-system` 스킬 또는 일반 흐름
- DB 마이그레이션 작업 — `db-squash` 스킬

## 유지보수

- 새 문서 추가·파일명 변경 시 갱신할 곳은 **`web/CLAUDE.md` 표 한 곳뿐** — 이 스킬은 손댈 필요 없음.
