# design.md 작성 참조 (Template · 참조형)

> ⚠️ **이 파일은 스킬에 번들된 제네릭 참조(템플릿)다. 특정 프로젝트의 규칙을 적지 않는다.**
> **프로젝트의 실제 디자인 규칙은 프로젝트의 `design.md`(권장: `docs/design.md`)에 둔다.**
> design-system 스킬은 이 템플릿을 보고 프로젝트 design.md 를 **생성/유지**한다.

---

## 역할 분리 (3 레이어)

| 레이어 | 역할 | 파일 |
|---|---|---|
| **스킬(방법)** | 토큰 탐지·분기·갱신 절차 | `SKILL.md` |
| **참조(템플릿/참조형)** | design.md 를 *어떻게* 쓰는가 — 포맷·예시·작성 규칙 | **이 파일** |
| **프로젝트(내용)** | 실제 색/타이포/간격/컴포넌트 규칙 | 프로젝트 `design.md` (예: `docs/design.md`) |

> 값(hex/oklch/px)은 코드 토큰(globals.css/tailwind/theme)에만 둔다. design.md 는 **토큰명·유틸명만** 참조한다. 충돌 시 **코드 토큰이 승리**하고, design.md 측 값은 제거한다.

---

## 포맷 A — 규칙 전용 (토큰 소스가 있을 때 · 권장)

값을 적지 않고 토큰명만 참조한다. (아래는 채움 예시 — 실제 토큰명은 프로젝트 토큰 소스에서 가져온다)

```markdown
# 디자인 규칙: {프로젝트명}

> 토큰 소스: {예: src/app/globals.css, tailwind.config.ts}
> 기준 화면(선택): {대표 화면 경로 — 톤·합성 기준}
> 마지막 업데이트: {YYYY-MM-DD}
> 규칙: 값(hex/px) 금지, 토큰명만. 코드와 충돌 시 코드가 승리.

## 0. 한 줄 방향성
{크리에이티브 한 줄}

## 1. 색상 사용 규칙
- `{token-primary}` — 주요 CTA에만, 페이지당 1~2회
- `{token-accent}` — 호버/하이라이트, 큰 면적 배경 금지
- `{token-semantic-*}` — 도메인 시멘틱(상태/위험 등)의 전용 용도
- (프로젝트 고유 주의사항이 있으면 명시: 예 "브랜드색과 특정 시멘틱색이 같은 계열")

## 2. 타이포 사용 규칙
- `{font-display}` — 헤드라인/제호만, 본문 금지
- 라벨 — 영문 한정 `uppercase` + `tracking`(언어별 주의)
- 수치 — `font-mono` + `tabular-nums`

## 3. 간격 · 레이아웃 규칙
- 컨테이너 폭/패딩, 섹션 간격, 모서리, 그리드 컨벤션

## 4. 컴포넌트 · 유틸 사용 규칙
- 카드/버튼/배지/입력의 표준 합성 공식, 강조 위계, 상태(빈/준비중/편집)

### Do / Don't
- ✅ 기존 토큰·유틸만 사용
- ✅ 새 토큰은 사용자 승인 후 코드 토큰 파일에 추가
- ❌ 인라인 hex/px 금지 — 토큰 경유
- ❌ design.md 에 토큰 값 적기 금지 — 토큰명만
```

## 포맷 B — 전체 (토큰 소스가 없을 때만)

토큰 파일이 전혀 없을 때만 값+규칙을 함께 정의한다. 첫 코드 작성 시 globals.css 등으로 **승격**한다.

```markdown
# 디자인 시스템: {프로젝트명}

> 토큰 소스: 없음 (이 문서가 값+규칙 모두 정의 — 첫 코드 작성 시 토큰 파일로 승격 예정)
> 마지막 업데이트: {YYYY-MM-DD}

## 방향성
{한 줄 요약}

## 색상
| 토큰 | 값 | 용도 |
|---|---|---|
| primary | #__ | 주요 CTA |
| bg | #__ | 페이지 배경 |
| surface | #__ | 카드 |

## 타이포
| 용도 | 폰트 | 크기 | 굵기 |
|---|---|---|---|
| Display | __ | __ | __ |
| Body | __ | __ | __ |

## 간격
| 토큰 | 값 |
|---|---|
| sm | __ |
| md | __ |
| lg | __ |

## 사용 규칙
(포맷 A 의 규칙 섹션과 동일)
```

## 작성 Do / Don't (이 참조를 쓰는 쪽)

- ✅ 프로젝트 규칙은 **프로젝트 design.md**(docs/design.md 등)에 쓴다. 이 참조 파일에는 쓰지 않는다.
- ✅ 토큰 소스가 있으면 **포맷 A(규칙 전용)** 를 쓴다.
- ✅ 갱신은 **변경된 섹션만** 수정한다(전체 재작성 금지).
- ❌ 이 참조 파일에 특정 프로젝트(브랜드색·폰트 등) 내용을 박아 넣지 않는다 — portable 유지.
- ❌ design.md(프로젝트/참조 모두)에 토큰 값(hex/oklch/px)을 적지 않는다.
