# 커맨드 사용설명서

이 프로젝트는 세 가지 슬래시 커맨드를 통해 **기획 → 개발** 워크플로우를 지원합니다.

---

## `/pm` — 기획/디자인

기능 아이디어나 UI 개선 요청을 기획 산출물로 만들어줍니다.

**실행 방식:** Foreground Agent (대화형 Q&A)

```
/pm 로그인 페이지 만들자
/pm 대시보드 다크모드 적용
/pm 사이드바 레이아웃 개선
```

### 동작 흐름

1. 요청 유형 자동 감지 (신규 기능 / 기존 개선)
2. 사용자와 Q&A로 요구사항 구체화
3. 산출물 생성:
   - **신규 기능** → `idea.html` (와이어프레임) + `brief.md`
   - **기존 개선** → `design.html` (Before/After) + `brief.md`
4. 사용자 승인 후 다음 단계 안내

### 산출물 위치

```
output/tasks/[ID]-[slug]/
├── idea.html 또는 design.html   ← 브라우저에서 확인
└── brief.md                      ← dev 커맨드의 입력물
```

---

## `/dev` — 구현

`/pm`이 만든 `brief.md`를 읽고 자동으로 구현합니다.

**실행 방식:** Background Agent (백그라운드 실행)

```
/dev 008-login
/dev 009-dashboard-redesign
```

### 전제 조건

- 해당 태스크 폴더에 `brief.md`가 존재해야 합니다.
- 없으면 `/pm`으로 먼저 기획하라는 안내가 나옵니다.

### 동작 흐름

1. `brief.md` + 프로젝트 코드 분석 (범위 파악)
2. 구현 계획 수립
3. 코드 구현 (프론트엔드 + 백엔드)
4. 에러 체크 (정적 분석, 빌드, 테스트)
5. `complete.md` 작성 + `history.json` 업데이트

### 산출물

```
output/tasks/[ID]-[slug]/
├── brief.md       ← 입력 (pm이 생성)
└── complete.md    ← 출력 (변경사항 + 에러체크 결과)
```

---

## `/do` — 직접 위임

기획이 필요 없는 간단한 작업을 바로 실행합니다.

**실행 방식:** Background Agent (백그라운드 실행)

```
/do 대시보드 카드 클릭 시 에러 발생
/do 사이드바 메뉴 텍스트 "프로젝트" → "워크스페이스"로 변경
/do C:/screenshots/bug.png 참고해서 수정
```

### 적합한 작업

- 버그 수정
- 텍스트/라벨 변경
- 간단한 UI 수정
- 스크린샷 기반 수정

### 동작 흐름

1. 태스크 ID 자동 채번 + 폴더 생성
2. 범위 파악 → 구현 → 에러 체크 (한 번에 진행)
3. `complete.md` 작성 + `history.json` 업데이트

---

## 일반적인 워크플로우

```
┌─────────┐     ┌─────────┐     ┌──────────────┐
│   /pm   │ ──→ │  /dev   │ ──→ │  완료 확인   │
│ 기획/QA │     │  구현   │     │ complete.md  │
└─────────┘     └─────────┘     └──────────────┘

┌─────────┐     ┌──────────────┐
│   /do   │ ──→ │  완료 확인   │
│ 직접위임│     │ complete.md  │
└─────────┘     └──────────────┘
```

| 상황 | 사용할 커맨드 |
|------|--------------|
| 새 기능을 기획부터 하고 싶다 | `/pm` → `/dev` |
| 기획서가 이미 있고 구현만 하고 싶다 | `/dev [태스크ID]` |
| 버그 수정, 간단한 변경 | `/do [설명]` |

## 태스크 관리

- 모든 태스크는 `output/tasks/` 아래에 폴더로 관리됩니다.
- `output/history.json`에서 전체 태스크 이력을 확인할 수 있습니다.
- ID 형식: `[3자리숫자]-[slug]` (예: `008-login`, `009-dashboard-redesign`)
